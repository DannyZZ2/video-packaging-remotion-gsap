# 关键词动效库 / Keyword Animation Effects

> 来源：用户收集的动效设计截图整理，包括“关键词动效”表中的弹出、放大、变色、高亮条、描边、敲章、震动、打字机、拆字、手势触发、词条贴纸、擦除/替换等基础动效。  
> 用途：在使用 `$video-use` 进行包装动效设计时，根据字幕语义、句子节奏和画面安全区选择合适的关键词动效。  
> 注意：原图右下角有遮挡，最后两项的“使用建议”为根据上下文补全，需用户确认。

## 使用原则

1. 先看字幕语义，再选动效：观点、结论、痛点、反差、纠错、步骤、工具名分别对应不同动效。
2. 关键词动效只强调“少数重点词”，不要让整条视频一直动。
3. 口播视频优先避开人脸、嘴部和底部中间字幕安全区。
4. 一条视频固定 1-2 种强调色，避免颜色过花。
5. 动效需要对齐发音节奏：关键词出现或高亮应落在对应词被说出的瞬间附近。
6. 除了文字动效，也可以选择卡片、鼠标、碰撞、拖拽、吸附等“交互感动效”，让包装更像真实操作而不是单纯字幕强调。
7. 不要生成表示整条视频播放进度的顶部/底部全局进度条。只有卡片内部的任务进度、加载条、步骤进度可以使用。
8. 本文件中的基础关键词动效是默认候选库；除非用户明确要求其它风格，不要用泛泛的“淡入淡出”替代这些可感知的强调方式。
9. 所有连线、箭头、路径、流程链路类动效，线条中间必须有一个沿路径移动的发光原点或光点，用来显性表现关联、传输、流向或数据正在移动。

## 动效清单

| 特效 | 画面表现 | 适用范围 | 使用建议 |
|---|---|---|---|
| 关键词弹出 | 关键词从人物旁边或字幕上方轻弹出来 | 观点词、结论词、痛点词、反差词 | 适合短词，1-4 个字最好；不要挡脸 |
| 关键词放大 | 字幕里的某个词突然变大、加粗 | 强调重点、讲到核心判断时 | 可配轻微音效，放大后快速回到正常大小 |
| 关键词变色 | 普通字幕保持白色，关键词变成黄、蓝、红等强调色 | 需要观众一眼抓重点的句子 | 一条视频固定 1-2 种强调色，避免花 |
| 关键词高亮条 | 词底部刷出荧光笔色块 | 方法论、清单、教程步骤、定义解释 | 适合偏干货、偏教学的段落 |
| 关键词描边 | 关键词加粗描边，形成短视频大字感 | 开头钩子、强观点、反转句 | 适合强情绪，不适合太长的句子 |
| 关键词敲章 | 关键词像印章一样盖上去，如“重点”“避坑”“错误” | 判断、提醒、踩坑、结论 | 很适合“别这么做”“真正的问题是”这类句子 |
| 关键词震动 | 词出现后轻微抖动 | 崩了、翻车、踩坑、爆了、卡住等情绪词 | 只用于情绪点，别让整条视频一直抖 |
| 打字机关键词 | 关键词逐字输入 | prompt、工具名、公式、结论句 | AI 教程里尤其适合展示 prompt 和命令 |
| 关键词拆字 | 一个词拆成几个字依次飞入 | 自动化、工作流、知识库、AI Agent 等概念词 | 偏炫酷，适合开头或章节标题 |
| 手势触发词 | 人物手势附近出现词条 | 真人口播、半身讲解、人物旁边有空白时 | 需要和手势方向匹配，避免看起来乱飘 |
| 词条贴纸 | 关键词做成小标签、贴纸、胶囊条 | 工具名、方法名、角色名、标签分类 | 图片遮挡，建议补全：适合并列信息和分类标签，一屏不要超过 2-3 个 |
| 关键词擦除/替换 | 旧词被划掉，新词替换出现 | 纠错、改认知、前后对比 | 图片遮挡，建议补全：适合纠偏、反转和前后对比；旧词弱化，新词更醒目 |

## 趣味交互动效 / Playful Interaction Effects

| 特效 | 画面表现 | 适用范围 | 使用建议 |
|---|---|---|---|
| 卡片弹跳入场 | 信息卡片从画面边缘滑入，落点轻微回弹 1-2 次 | 新概念、新工具、新结论第一次出现 | 弹跳幅度要小，落点避开脸部；适合 1 张主卡或 2 张小卡 |
| 卡片点击展开 | 鼠标指针点击卡片，卡片从小标签展开成信息面板 | 展示工具功能、方法解释、步骤详情 | 点击点要清楚，展开后停留足够阅读；不要遮挡嘴部 |
| 鼠标点击高亮 | 鼠标移动到关键词或按钮上，点击后出现光圈、高亮或勾选状态 | prompt、按钮、工具名、操作步骤 | 适合 AI 教程和软件演示；点击光圈 6-10 帧即可，不要一直闪 |
| 卡片碰撞 | 两张卡片从两侧滑入轻碰，碰撞后弹开或合并成结论卡 | 对比、冲突、反差、旧方案 vs 新方案 | 碰撞要有轻微 squash/stretch；适合表达“对比后得出结论” |
| 卡片吸附归位 | 多个小标签飞入后自动吸附到一个面板或流程图节点 | 分类、归纳、知识库、工作流整理 | 每次只吸附 1 个元素，错峰进入，避免同时飞太多 |
| 卡片堆叠展开 | 一摞卡片从背后依次错位展开，露出 3-4 个关键词 | 清单、步骤、功能列表、多个能力点 | 适合“它能做这些事”；最多 4 张，超过会显乱 |
| 卡片翻转 | 卡片正面是旧认知，翻转后背面变成新结论 | 纠错、反转、认知升级 | 翻转前后文字必须短；翻转轴固定，避免空间感混乱 |
| 拖拽排序 | 鼠标拖动几个标签重新排序，最后形成清晰流程 | 工作流、优先级、步骤重排 | 适合“从零散到有序”的表达；拖拽路径要短 |
| 磁吸连线 | 卡片靠近后自动生成连线，形成节点关系 | Agent、自动化链路、工具协作、流程关系 | 线条从节点边缘生长，不穿过人物脸部 |
| 勾选完成 | 卡片左侧依次出现 checkbox 勾选，配合轻微弹动 | 清单完成、步骤推进、任务执行 | 适合教程、任务流；勾选节奏跟语音列举同步 |
| 进度条加载 | 卡片底部或任务行内部出现加载条，推进到完成状态 | 生成、分析、渲染、处理中 | 只用于卡片内部任务状态；不要做成整条视频的顶部/底部全局进度条 |
| 错误抖动纠正 | 错误卡片左右抖动，随后被替换为正确卡片 | 错误示范、避坑、纠错 | 抖动 2-3 次即可，随后快速收束，避免眩晕 |
| 图标飞入卡片 | code、video、agent、prompt 等图标先飞入，再带出文字卡 | 工具组合、能力拆解、模块介绍 | 图标比文字先出现 4-8 帧，帮助观众预判信息类别 |
| 便签贴上墙 | 小便签从下方或侧边贴到画面空白区域，角度轻微随机 | 灵感、备注、经验、注意事项 | 适合轻松口吻；角度控制在 -4 到 4 度，不要太乱 |
| 聚光灯扫过 | 局部光带扫过卡片或关键词，只照亮重点词 | 重点揭示、结论落点 | 只扫卡片内部，不做全屏扫光；如果风格禁止转场光效，仍可用局部卡片内高亮 |

## Remotion 原生信息图动效 / Remotion-Native Infographic Effects

> 来源：已确认的默认风格样张和 Remotion 预览。这里只记录动画、呈现方式和适用场景，不记录具体内容。

| 特效 | 画面表现 | 适用范围 | 使用建议 |
|---|---|---|---|
| 模块卡片入场 | 半哑光深色卡片从侧边或下方浮入，图标、英文标签、中文关键词错峰出现 | 工具名、步骤、状态、能力点 | 一屏 1 张主卡 + 1-2 个小标签；适合放左侧或右侧空区 |
| 动态关键词排版 | 大号关键词用遮罩揭示，pill 高亮从词中心扩张，下划线绘制 | 观点词、结论词、操作动作、反问词 | 主词必须短；不把整句字幕做成大字 |
| 信息图环绕卡 | 源节点分出 2-3 条曲线到目标卡，目标卡按语义逐个点亮 | 流程路径、方案分支、概念关系、工具协作 | 曲线从节点边缘连接到目标卡边缘；每条线必须有循环移动光晕点 |
| 前后替换卡 | 旧词变暗或划掉，箭头指向新词，新词在箭头目标点弹入 | 纠错、改认知、旧方案到新方案 | 旧词和新词一屏最多 2 组；箭头不能错位 |
| 鼠标点击标注 | 指针移动到卡片或关键词，点击后出现 6-10 帧光圈和状态变化 | prompt、按钮、工具名、操作步骤 | 点击点要清楚；光圈只在局部，不做全屏闪烁 |
| 短终端标签 | 短命令条或状态 chip 打字进入，作为局部说明 | prompt、命令、部署、调用 | 只展示短命令，不做大面积终端窗口 |
| 磁吸标签流 | 小标签从画面边缘飞入并吸附到主卡或节点上 | 分类、归纳、工作流整理 | 每次只吸附 1 个元素，错峰 4-8 帧 |

### Remotion 原生组件可调用规范 / Callable Remotion-Native Specs

> 给 `$video-use` 生成动效方案时，可直接引用下面的 `effectId`。每个效果仍必须绑定字幕关键词落点；不要按均分时间触发。
> When `$video-use` drafts packaging motion, it can reference the `effectId` below directly. Every effect still must be anchored to a subtitle keyword cue, not an evenly split timeline.

| effectId | 中文名 | 触发关键词类型 | 推荐层级 | 动画步骤 | 安全区规则 |
|---|---|---|---|---|---|
| `native.modularCard` | 模块卡片 | 工具、步骤、状态、能力、输出 | 视频背景 -> 轻遮罩 -> 卡片主体 -> 图标容器 -> 标签和关键词 | 卡片 y 位移 + opacity 入场；图标先 pop；英文标签和中文关键词错峰出现；边框弱发光一次 | 放左侧、右侧或上半区；宽度不超过画面 32%；避开脸和嘴 |
| `native.kineticTypeCard` | 动态关键词排版 | 重点、结论、点击、生成、修复、钩子 | 视频背景 -> 关键词文字 -> pill 高亮 -> 下划线 -> 小贴纸 | 文字用 `clip-path` 揭示；pill 从中心扩张；下划线 `scaleX` 绘制；贴纸旋转回正 | 主词最多 1-4 个字或 1-2 个英文词；不放字幕安全区中心 |
| `native.orbitFlow` | 信息图环绕卡 | 流程、分支、去向、路径、协作 | 视频背景 -> 源节点 -> SVG 曲线 -> 移动光点 -> 目标卡 | 源节点先出现；曲线逐条 `strokeDashoffset` 绘制；光点沿线循环；目标卡在光点接近时点亮 | 曲线不能穿过脸；穿过人物时必须放人物遮罩下方 |
| `native.comparisonSwap` | 前后替换卡 | 替换、纠错、改成、不要、应该 | 视频背景 -> 旧词卡 -> 箭头 -> 新词卡 -> 结果高亮 | 旧词轻震或划掉；箭头 8-12 帧绘制；新词在箭头目标处弹入 | 新词必须在箭头指向位置；不能贴边或压字幕 |
| `native.mouseClickCallout` | 鼠标点击标注 | 点击、选择、打开、输入、执行 | 视频背景 -> 指针 -> 点击光圈 -> 局部高亮 -> 状态 chip | 指针移动到目标；点击时光圈扩散；目标卡边框亮 6-10 帧 | 指针和光圈只做局部强调，不遮挡脸和嘴 |
| `native.miniTerminalTag` | 短终端标签 | prompt、命令、安装、部署、调用 | 视频背景 -> 短命令条 -> 光标 -> 状态标签 | 命令短打字机进入；光标闪 1-2 次；状态标签 pop | 命令不超过一行；不使用大面积终端窗口 |

连线类补充规则：

- `native.orbitFlow` 的每条曲线都要有一个发光原点沿曲线移动，路径颜色与对应目标卡一致。
- `native.comparisonSwap` 的旧词到新词之间，需要有箭头线和移动光点，表示替换/修正方向。
- 其它任何 `line/path/arrow/connector` 组件，也按同样规则加移动光点；光点必须是柔和光晕，不是生硬实心圆，大小控制在 6-12px，外发光半径不超过 28px，避免变成转场光效。
- 连线光点必须沿线条循环运动。路径画完后不能停在终点不动；可用 `loop01(frame, start, duration)`、`MotionPathPlugin`、SVG `getPointAtLength` 或等价方法让光晕持续沿路径移动。
- 连线长度必须根据目标元素边框和内容宽度计算：起点贴合源节点边缘，终点贴合目标卡片/标签边缘，不能靠固定长度目测。
- 替换/纠错类卡片中，新词必须放在箭头指向的目标位置，不能出现在旧词下方或其它非指向区域。
- 环形仪表、进度弧在数值为 0 时不能露出彩色进度端点；应使用 `pathLength`、`strokeDasharray`、`clipPath` 或显式 `opacity` 处理 0 值状态。

## 选择建议

| 字幕语义 | 优先选择 | 避免 |
|---|---|---|
| 观点、结论、痛点、反差 | 关键词弹出、关键词描边、关键词敲章 | 太长的整句大字 |
| 核心判断、重点强调 | 关键词放大、关键词变色 | 连续多个词同时放大 |
| 方法论、清单、教程步骤 | 关键词高亮条、词条贴纸、勾选完成、卡片堆叠展开 | 强震动、过度炫酷拆字 |
| AI prompt、命令、公式、工具名 | 打字机关键词、鼠标点击高亮、卡片点击展开 | 随机弹跳或花哨变色 |
| 自动化、工作流、知识库、Agent 概念 | 关键词拆字、磁吸连线、卡片吸附归位、拖拽排序 | 过长词组拆字 |
| 踩坑、错误、崩了、翻车 | 关键词震动、关键词敲章、错误抖动纠正 | 长时间持续抖动 |
| 纠错、改认知、前后对比 | 关键词擦除/替换、卡片翻转、卡片碰撞 | 只变色但没有前后关系 |
| 真人手势、指向、旁边有空白 | 手势触发词、词条贴纸、便签贴上墙 | 词条飘到脸部或嘴部 |
| 工具能力、模块组合 | 图标飞入卡片、卡片弹跳入场、卡片点击展开 | 一次出现太多卡片 |
| 生成中、处理中、任务执行 | 进度条加载、勾选完成、鼠标点击高亮 | 长时间无变化的加载条 |

## 给 video-use 的动效设计要求

在生成包装方案时，每个关键词动效至少写清楚：

- 时间：开始时间、结束时间、对应字幕句子。
- 文字：屏幕上出现的关键词或短语。
- 类型：从本动效库中选择的特效名称。
- 动作：入场、停留、强调、退场或替换方式。
- 交互：如鼠标点击、卡片碰撞、拖拽、吸附、勾选、加载等具体交互行为；没有交互则写“无”。
- 布局：放在左侧、右侧、上半区或字幕上方，明确避开脸部和嘴部。
- 风格：颜色、描边、阴影、胶囊条、贴纸、字体粗细等。

示例格式：

```text
00:03.20-00:04.40｜关键词：真正的问题｜特效：关键词敲章
布局：人物左侧上半区，不挡脸；红色半透明印章框，文字白色加粗。
动效：在“真正的问题”说出口时 6 帧缩放敲入，轻微旋转 -3 度，落地后 0.8 秒保持，随后透明度淡出。
```

```text
00:06.80-00:08.20｜关键词：自动工作流｜特效：卡片吸附归位
布局：人物左侧中上区，3 个小标签从左侧依次飞入，吸附到一个主流程卡片上。
动效：每个标签错峰 8 帧进入，落点轻微回弹；最后生成一条青色连线，主卡片发光 0.4 秒。
```

```text
00:10.40-00:11.70｜关键词：不是网页工具｜特效：卡片碰撞 + 卡片翻转
布局：左侧安全区，两张小卡分别写“网页工具”和“Agent 工作流”。
动效：“网页工具”卡从左滑入，“Agent 工作流”卡从右滑入轻碰；碰撞后旧卡透明降低，新卡翻转放大成为主卡。
```
